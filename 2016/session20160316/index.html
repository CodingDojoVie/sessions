Classifieds Specs

  • A user has to provide all the information, it has to be valid
  • If the user provides non-valid information, all false inputs have to be mentioned with a user-friendly message
  • Write a validation-framework for it


Fields + User Validation Rules

  • heading: at least 5 char, not more then 40
  • description: at least 20 char, not more then 1000
  • name: at least 5 char, not more then 40
  • phonenumber: only numeric, at least 8 numbers, at most 20
  • category: you can choose how to categorize a classified, but it has to be categorized, it's not allowed to leave it uncategorized

Modified Specs A

  • Users can create real-estate ads
    • heading: no input, generated from location and space
    • description: same as classifieds
    • name: same as classifieds
    • phonenumber: same as classifieds
    • location: at least 10 char, at most 100 char
    • space: in square meter, at least 1 squaremeter, at most 100000 squaremeter


Modified Specs B

  • There is an importer, it receives this information from an xml file. Same fields, but different validation (phonenumber can be plain text, ...)
  • admins on the admin interface can modify all fields at free will, except categories
  • users can only edit their own ads
