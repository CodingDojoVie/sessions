The TDD Cycle Revisited:
Fail fast and in Beauty


  1. Start with a unit test and make it fail great:
    • Name of the test
    • Assertion message
  2. Make the test pass by writing production code
    => "Do the simplest thing that could possibly work."
  3. Refactor both the test and the production code.
