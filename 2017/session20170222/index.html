What is a Coding Dojo?


  • Safe place outside of work and away from production code
  • Deliberate Practice => Learning
  • Relax and Slow down
  • You don't have to finish
  • Focus on doing it right

Open Closed Principle

The open/closed principle states

software entities (classes, modules, functions, etc.) should be open for extension, but closed for modification

− Bertrand Meyer (1988)

Requirement #1

A number should be returned as a string.


Example

1 ==> "1"

2 ==> "2"

Workflow

  1. Write the first failing test.
  2. Then write a factory that returns an object, or an aggregate of objects, that make the test pass.


The factory should be limited to creating objects and linking them together. No conditionals allowed.

Requirement #2

A number that is divisible by 3 should be returned as the word Fizz.


Example

3 ==> "Fizz"

12 ==> "Fizz"

Workflow

  1. Write the next failing test.
  2. Can you make it pass by changing the factory and/or creating a new class and nothing else?

What we're not solving

  • We will not check for valid rolls.
  • We will not check for correct number of rolls and frames.
  • We will not provide scores for intermediate frames.

Simple bowling rules

A game has 10 frames (2 rolls each).

Spare

10 pts + score of next roll

Strike

10 pts + sum of score of next two rolls

10th Frame

Total number of pins knocked down for all rolls (spare/strikes do not give any extra points)

First test

Rolling zero each time should result in the score being zero (0) at the end of the game.
